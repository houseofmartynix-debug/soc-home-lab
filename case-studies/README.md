# Case Studies

These are **simulated investigations** I ran in my home lab — each one starts with an alert generated by a real attack (run from the Kali VM), and walks through the exact steps I took to triage, investigate, and close the case.

The aim is to show that I can think like a SOC analyst, not just write rules.

## Index

| Case | Trigger | Outcome |
|---|---|---|
| [CASE-001](case-001-ssh-bruteforce.md) | SSH brute force from external IP | Contained — IP blocked at perimeter |
| [CASE-002](case-002-suspicious-powershell.md) | Encoded PowerShell from Word macro | True positive — host reimaged |
| [CASE-003](case-003-phishing-investigation.md) | User-reported phishing email | Campaign-wide purge |

## Format

Each case follows the same structure for consistency:

1. **Alert** — what the SIEM showed
2. **Triage** — first 5 minutes
3. **Investigation** — what I pivoted on, what I found
4. **Response** — actions taken
5. **Closure & Lessons** — disposition + what I'd improve

## What These Don't Show

I deliberately used fake hostnames (`WIN10-01`, `UBUNTU-30`) and a private RFC1918 + a placeholder external IP. No real data is included.
